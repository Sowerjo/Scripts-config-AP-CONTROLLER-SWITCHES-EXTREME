- Recomenda-se configurar como modo de espera para o cluster secundário.

CONFIGURANDO ROAMING ASSISTIDO

Procedimento
A configuração 802.11v bss-transition é suportada na CLI apenas atualmente.

Este comando precisa ser digitado completamente na CLI a partir de agora - em wlan "802.11v bss-transition".


Crie e use a política de assistência de roaming para ajudar com clientes fixos - roaming-assist-policy <nome> e defina condições para invocá-la.
E então em wlan "use roaming-assist-policy <nome>.


WLC>en
WLC#config
WLC(config)#wlan test
WLC(config-wlan-test)#radio-resource-measurement
WLC(config-wlan-test)#802.11v bss-transition

WLC>en
WLC#config
WLC(config)#roaming-assist-policy test-roam
WLC(config-roaming-assist-policy-test-roam)#?

Roaming Assist Mode commands:
  action               Configure action - action is deauth / log /
                       assisted-roam
  aggressiveness       Configure the roaming aggressiveness for a wireless
                       client
  detection-threshold  Configure the detection threshold - when exceeded,
                       client monitoring starts
  disassoc-time        Configure the disassociation time - time after which a
                       disassociation is sent
  handoff-count        Configure the handoff count - number of times client
                       can exceed handoff threshold
  handoff-threshold    Configure the handoff threshold - when exceeds an
                       action is taken.
  monitoring-interval  Configure the monitoring interval - interval at which
                       client monitoring occurs
  no                   Negate a command or set its defaults
  sampling-interval    Configure the sampling interval - interval at which
                       client rssi values are checked
WLC(config)#wlan test
WLC(config-wlan-test)#use roaming-assist-policy test-roam
WLC(config-wlan-test)#comit write


Notas adicionais
O 802.11k deve estar habilitado no dispositivo para que o 802.11v funcione.
O 802.11v precisa ser suportado e habilitado no cliente para funcionar.

BLOQUEAR MAC ADDRESS ESPECIFICO DA REDE

       1. Create an Association Policy.  Each rule in the policy will be for an individual MAC address or a range of MAC addresses. There is an implicit deny at the end so if you want to allow all others you need to add a permit all at the end (as seen below.) 
 
!
association-acl-policy Test
deny 7C-50-49-57-A6-81 precedence 1
permit 00-00-00-00-00-00 FF-FF-FF-FF-FF-FF precedence 1000
!
 
      2. Apply it to a WLAN

 !CON
wlan Test_SSID
use association-acl-policy Test
!

     3. Commit and save



enable
config t
association-acl-policy ACL_BLOCK_USERS_MOBILE
deny 7C-50-49-57-A6-82 precedence 1
com wr
